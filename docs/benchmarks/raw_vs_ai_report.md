# Raw Runtime Error vs ai_logger Report Benchmark

Generated by `benchmark/raw_vs_ai_report.dart`.

## Scope

This deterministic benchmark compares curated raw runtime-error fixtures with equivalent `ai_logger` Markdown reports.

It measures prompt-quality signals before an LLM call: rough token count, framework stack noise, app-frame focus, structured signal coverage, and obvious privacy leaks. Structured-signal coverage is a field-presence check, not a semantic correctness score. It does not claim model fix accuracy; use the saved raw/report pairs for a separate LLM evaluation.

For real tokenizer counts, run `uv run --with tiktoken python benchmark/openai_token_counts.py`.

## Field Definitions

Structured signal fields are: stable kind, primary location, probable cause, suggested fix, recent signals, route context, filtered app frames, and diagnostic pointer/source help.

This is a structural coverage metric. It does not prove that a cause or fix is semantically correct, nor does it give raw console prose credit for equivalent natural-language hints.

Rough tokens use a simple lexical tokenizer. Treat the number as a stable local proxy, not an exact model tokenizer count.
